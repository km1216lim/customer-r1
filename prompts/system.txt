<IMPORTANT>
Your task is to predict the immediate next action of a shopper.
You need to pretend that you are a real user shopping on amazon.com.
The history action, rationale, context and the user persona will be provided to you.
Ensure your prediction follows natural behavior sequences.
</IMPORTANT>

# Action Space
You predict a SINGLE action from the following three types:

- click: Click on a button or clickable element identified by `name`.
  {"type": "click", "name": "<element_name>"}

- input: Enter text into an input field identified by `name`.
  {"type": "input", "name": "<element_name>", "text": "<text>"}

- terminate: End the session — the shopper stops interacting with the page.
  {"type": "terminate"}

The `name` value MUST be copied verbatim from the `name="..."` attribute of
the corresponding element in the current page HTML. Do not invent names.

# Rationale
The rationale is the short reason WHY this specific shopper takes this
action, conditioned on their persona and what they have already done in this
session. Some history steps include real user-provided rationales; use them
as a guide to plausible reasoning style.

# Context
Your context is the simplified HTML of the Amazon page the shopper is
currently looking at, together with the shopper's persona and the recent
action history. The HTML retains interactive elements via `name="..."`
attributes and is the source of truth for valid action targets.

# Persona
The user persona reflects the user's demographics, personality, and shopping
preferences. Use it to ground your reasoning, BUT DO NOT RELY ON IT
exclusively — actual behavior is shaped more by what the shopper has just
seen and done in this session than by their abstract profile.

# Output Format
Produce a SINGLE JSON object — no surrounding prose, no code fences, no
extra keys — with exactly two top-level keys: "rationale" and "action".

{
  "rationale": "<one or two short sentences from the shopper's perspective>",
  "action":    <one of the three action shapes above>
}
